xFOSE v1.4 Release Notes
========================

New Features
------------

**ScriptInterface API**
- New interface (kInterface_Script) for calling game scripts from plugins
- UDFCaller::CallUDF() enables dynamic script execution with parameter passing and return values
- FindFirstScript export for locating suitable scripts in game data
- Supports quest scripts and object scripts with automatic type selection

**ScriptRunner::ExecuteLine Fix**
- Fixed calling convention: changed to __fastcall with ECX=this + 9 stack params
- Corrected parameter layout based on captured game calls via hook instrumentation
- p2 = block body offset (parsed from Begin block header)
- p5 = Begin chunk data length (parsed from bytecode)
- p7 = execution mode flag (2)
- p4/p6 = adjacent UInt32* for opcode offset tracking
- Zeroed ScriptRunner is sufficient (no complex initialization needed)
- No more stack corruption or crashes when calling scripts via FOSE

Technical Details
----------------
- Hook instrumentation added to capture game's natural ExecuteLine calls
- Used captured data to reverse-engineer correct parameter layout
- Bytecode parsing added to dynamically extract block offsets
- Tested in-game with quest scripts — ExecuteLine returns success (1) consistently

Backward Compatibility
----------------------
- 100% backward compatible with existing FOSE plugins
- Drop-in replacement for FOSE 1.7
- No changes to existing Event Manager, DataInterface, or other APIs

Installation
------------
1. Extract fose_loader.exe and fose_1_7.dll to your Fallout 3 directory
2. Launch the game using fose_loader.exe
3. Existing plugins will continue to work as before
